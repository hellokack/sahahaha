## 슬라이드: 1단계 - 완료된 작업

### 데이터 파이프라인 구축

- 사하구청 홈페이지 6개 메뉴 카테고리 크롤링
  - BFS 기반 페이지 탐색, 메뉴당 최대 50페이지
  - requests + BeautifulSoup, 동적 페이지는 Selenium 자동 전환
- 텍스트 정제: 네비게이션 잔재 제거, 500자 청크 분할 (50자 오버랩)
- Groq LLM으로 메타데이터 자동 태깅 (서비스 유형, 키워드, 대상 등)
- HuggingFace MiniLM-L12-v2로 384차원 벡터 임베딩 (로컬 CPU, 무료)
- Supabase pgvector에 벡터 저장

### 챗봇 엔진 구축

- RAG 기반 답변 생성 (Groq Llama 3.3 70B)
- 하이브리드 검색: 1차 메타데이터 필터링 → 2차 벡터 유사도 → 폴백 로직
- 멀티턴 대화: 최근 10개 대화 유지, 이전 키워드 포함 검색
- 개인정보 보호: 주민등록번호/전화번호/이메일 감지 시 LLM 전달 차단
- 출처 관련성 필터링: 키워드 매칭으로 무관한 출처 제거

### 자동화

- APScheduler로 매일 새벽 3시 증분 크롤링 자동 실행
- MD5 해시 비교로 변경된 페이지만 감지 → 해당 청크 재처리
- Quick Pipeline: API 할당량 부족 시 태깅 생략하고 임베딩만 수행

### 프론트엔드

- 반응형 채팅 UI (PC: 카드형 / 모바일: 전체화면)
- OS 설정 연동 다크모드 자동 적용
- 출처 URL 클릭 가능한 카드 UI
- 홈페이지 iframe 임베딩용 위젯 페이지

## 슬라이드: 2단계 - 코드 분석에서 도출된 개선 과제

### 보안 취약점 (발견 7건)

| 심각도 | 내용 | 위치 |
|:------:|------|------|
| 높음 | 벡터 검색 시 전체 테이블 풀스캔 (임베딩 데이터 네트워크 노출) | vector_store.py |
| 높음 | /api/stats 인증 없이 시스템 정보 노출 | app.py |
| 높음 | 에러 메시지에 내부 정보(DB URL 등) 포함 가능 | app.py |
| 높음 | Supabase RLS 미설정 시 전체 데이터 접근 가능 | database.py |
| 중간 | Flask Secret Key 기본값 하드코딩 (세션 위조 가능) | config.py |
| 중간 | CORS/CSP 헤더 미설정 (iframe 클릭재킹) | app.py |
| 중간 | API Rate Limiting 없음 (Groq 할당량 소진 위험) | app.py |

### 성능 병목

| 내용 | 영향도 |
|------|:------:|
| 매 검색마다 documents 테이블 전체 다운로드 후 Python 유사도 계산 | 높음 |
| /api/stats 호출마다 임베딩 모델 재로딩 (10초+) | 중간 |
| Supabase 클라이언트 중복 생성 (Database, VectorStore 각각) | 낮음 |
| 크롤러 BFS 큐 list.pop(0) → O(N) | 낮음 |

### 기능 보완 필요

- LLM 응답에 대한 개인정보 필터링 누락 (입력만 체크)
- 대화 이력 영구 적재 (만료 정책 없음)
- 역질문 감지가 하드코딩 키워드 방식 (LLM 응답 변형에 취약)
- 메타데이터 태깅 속도 (청크당 25초 → 100개에 42분)

## 슬라이드: 3단계 - 향후 계획

### 단기 (보안 패치 + 성능 최적화)

```
[현재]                              [개선 후]
Python에서 전체 문서 풀스캔     →   Supabase match_documents() RPC
매 검색 수 MB 네트워크 전송     →   서버사이드 pgvector 인덱스 검색
/api/stats 무인증              →   관리자 인증 + 에러 마스킹
Rate Limiting 없음             →   flask-limiter IP당 분당 10회 제한
Secret Key 하드코딩            →   환경변수 필수화
```

- 벡터 검색을 `match_documents()` RPC로 전환 (성능 + 보안 동시 해결)
- 관리자 API 인증 추가 + 에러 메시지 일반화
- flask-limiter로 Rate Limiting 적용
- Supabase RLS 정책 설정
- CORS/CSP 헤더 설정

### 중기 (비동기 서버 전환)

```
[현재]                              [전환 후]
Flask (동기, WSGI)              →   FastAPI (비동기, ASGI)
time.sleep() 블로킹             →   asyncio 기반 논블로킹
단일 요청 처리                  →   동시 다중 요청 처리
```

- Flask → FastAPI 전환으로 동시 처리 성능 향상
- Groq API 호출을 async/await로 비동기화
- Supabase 클라이언트를 비동기 버전으로 교체
- DB 커넥션 풀링 적용

### 장기 (기능 고도화)

- 답변 품질 피드백 시스템 (좋아요/싫어요 → 검색 정확도 개선)
- LLM 응답 개인정보 마스킹 (입력뿐 아니라 출력도 필터링)
- 대화 이력 TTL 정책 (30일 자동 삭제)
- 메타데이터 태깅 배치 처리 (다중 청크 → 1회 LLM 호출)
- 다국어 지원 (외국인 주민 대상 영어/중국어)
