## 2. UI 저장소 프론트엔드 통합

`sahahaha-team/UI`의 `chatbot-ui` 브랜치는 본 프로젝트와 **공통 조상이 없는 별도 히스토리**이며,
백엔드는 NER/BM25/증분 크롤링 이전의 구버전이었다. 따라서 단순 머지 대신 **프론트엔드만 선별 통합**.

| 가져온 파일 | 내용 |
|------|------|
| [templates/index.html](../templates/index.html) | 설정 메뉴(다크모드/글자크기), 마스코트 아바타 |
| [static/css/style.css](../static/css/style.css) | 다크모드/테마 변수, UI 리디자인 |
| [static/js/chat.js](../static/js/chat.js) | TTS(`speechSynthesis`), 마스코트 상태 전환 |
| static/images/mascot_{neutral,thinking,warning}.png | 마스코트 3종 |

- **제외**: 백엔드(app.py/chatbot/crawler 등), 문서, `sa_config.env`(실키 노출 파일)
- **호환 확인**: 새 `chat.js`는 `/api/chat`·`/api/clear`만 호출, 응답 필드(`answer`/`sources`/`degraded`) 일치
- **보안 경고**: UI 저장소 `sa_config.env`에 Groq/Supabase 실키가 노출되어 있어 폐기·재발급 권고

## 3. 통합 직후 결함 4건 수정 (`87ac0bb`)

### 3.1 위젯 회귀 — `/widget` 챗봇 동작 불능 (🔴)
새 `chat.js`가 설정 UI 요소(`btn-settings` 등)를 최상단에서 참조하는데,
[templates/widget.html](../templates/widget.html)에는 해당 요소가 없어 `TypeError`로
**스크립트 전체(전송/입력 바인딩 포함)가 중단**되던 문제.
→ 설정 관련 핸들러를 `if (el) { ... }` null-guard로 감쌈.

### 3.2 굵게 렌더링 — 정규식 오류
`/\*\*(.*?)\*\"/g` → `/\*\*(.*?)\*\*/g` 로 교정. `**굵게**`가 `<strong>`으로 정상 렌더.

### 3.3 PII 과잉 마스킹 — AF 라벨 제외
KoELECTRA NER이 '지하철/버스'를 `AF`(Artifact)로 오탐 → `[MASKED:물건]`으로 가려지던 문제.
[chatbot/pii_detector.py](../chatbot/pii_detector.py) `MASK_LABEL_MAP`에서 `AF` 제거(PS·LC만 유지).

### 3.4 출처 폴백
관련성 필터를 통과한 출처가 없을 때 상위 결과로 폴백하여 출처가 항상 제시되도록.
(※ 이후 §5.1에서 역질문 시에는 예외 처리로 보완)

## 4. 미팅 피드백 반영 (개선요청 3건)

### 4.1 PII 필터 행정용어 예외 (오탐 개선)
'옴부즈만' 같은 행정 제도어를 **사람 이름으로 오인해 답변 전체가 차단**되던 엣지 케이스.

- [chatbot/pii_detector.py](../chatbot/pii_detector.py) `SAFE_TOKENS`에 행정 제도·기관·직책 용어 추가
  - 옴부즈만/시민옴부즈만/감사관, 소통감사실/구청장/부구청장/동장/통장,
    주민센터/행정복지센터/민원실/구의회, 보건소/복지관/도서관/문화원 등
- 검증: "옴부즈만 제도가 뭔가요?" → 차단 없이 정상 답변 생성

### 4.2 출처 카드 구체화 — 부서명 + 연락처
출처에 담당 부서명(과)과 연락처 안내를 함께 제시.

- 신규 [chatbot/dept_directory.py](../chatbot/dept_directory.py): 부서 보정 + 연락처 디렉터리
- [chatbot/retriever.py](../chatbot/retriever.py): 출처에 `department`(보정)·`contact` 부여
- [app.py](../app.py) `Source` 모델에 `contact` 필드 추가
- [static/js/chat.js](../static/js/chat.js): 카드에 `🏛️ 담당: OOO과` + 하단 연락처 안내 문구
- **사실 기반 원칙**: 확인된 직통번호가 없으면 **대표전화(051-220-4000)로 폴백**.
  없는 번호를 지어내지 않음(`DEPT_CONTACTS`는 공식 번호 확인 시에만 등재)

### 4.3 부서명 오기 보정
'도로과'(실제 '도로정비과') 등 조직도에 없는 명칭 출력 사례 보정.

- [chatbot/dept_directory.py](../chatbot/dept_directory.py) `DEPT_CORRECTIONS = {"도로과": "도로정비과"}`
- 적용 지점 3곳: 답변 본문([conversation.py](../chatbot/conversation.py) `normalize_dept_names`),
  출처([retriever.py](../chatbot/retriever.py)), 태깅 단계([metadata_tagger.py](../processor/metadata_tagger.py))
- ⚠️ **한계**: 문자열 오기 보정만 가능. "자동차세→환경과"식 **의미적 오분류**는 미해결(§6 로드맵)

## 5. 역질문 출처 처리 + 연락처 위치 개선

### 5.1 역질문(되묻기) 응답에는 출처 비표시
"환경" 같은 모호한 질문에 옵션을 되묻는 화면인데도 §3.4 폴백이
**관련없는 복지 문서를 출처로 붙이던** 문제(미팅 화면 지적).

- [chatbot/conversation.py](../chatbot/conversation.py): `is_clarification`이면 `sources = []`
  (아직 '답변'이 아니라 질문을 되묻는 단계이므로 출처를 표시하지 않음)
- 검증: "환경" → 역질문·출처 0개 / "주민등록등본 발급" → 출처 2개(민원여권과) 유지

### 5.2 연락처 안내 위치 — 흰색 말풍선 하단으로
별도 파란 박스(출처 영역 아래) → **주요 답변(흰색 말풍선) 맨 아래에 점선 구분선과 함께 포함**.

- [static/js/chat.js](../static/js/chat.js): 연락처 생성 로직을 `createSourcesEl` → `createMessageEl`(말풍선 내부)로 이동
- [static/css/style.css](../static/css/style.css): `.source-contact`(박스) → `.bubble-contact`(말풍선 내 구분선) 스타일로 교체

## 6. 향후 로드맵 (미팅 §3·§4)

| 항목 | 성격 | 제안 |
|------|:---:|------|
| 부서명/연락처 정확도 근본 개선 | 데이터 | 홈페이지 **조직도 크롤링** → 부서명·직통번호 마스터 구축, 보정사전·`DEPT_CONTACTS` 자동 충진 |
| 동시접속 부하 테스트(100명+) | 검증 | locust/k6 부하 테스트 스크립트 작성 |
| PDF·HWP 파일 수집 | 기능확장 | 크롤러가 현재 `.pdf/.hwp` 제외 → 파서 추가 필요 |
| 음성 입력(STT) | 기능확장 | 브라우저 `SpeechRecognition` API (TTS는 구현 완료) |
| 공식 마스코트 '고니' | 외부의존 | 구청(도시디자인과) 에셋 수령 후 이미지 교체 |
| 표(Table) 출력 | 모델한계 | 프론트 마크다운 표 렌더링 추가 가능하나 모델 출력 품질이 관건 |
