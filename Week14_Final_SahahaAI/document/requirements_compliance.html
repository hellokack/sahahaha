요약 (Executive Summary)

요청하신 프로그램 형태 · 4대 주요 기능 · 3대 유의사항의 모든 항목을 구현·검증 완료했습니다. 특히 증분 크롤링, 2단계 하이브리드 검색, 멀티턴 역질문은 명세 요구 이상으로 고도화하였습니다.

요구사항상태구현 핵심
웹 애플리케이션 (홈페이지 연동형)충족FastAPI 웹앱 + iframe 임베드용 위젯 페이지
반응형 UI/UX (PC·모바일)충족반응형 레이아웃 + 다크모드·글자크기·TTS
① 증분 크롤링충족 ＋ETag 조건부 GET, 매일 자동 갱신
① AI 자동 태깅충족LLM으로 부서·분야·키워드 자동 추출
② 하이브리드 검색 (2단계)충족 ＋메타필터 → 벡터 → BM25 재랭킹

＋ 표시는 명세 요구를 넘어 추가 고도화한 항목입니다.

1. 프로그램 형태

웹 애플리케이션 — 홈페이지 연동형 충족

요구 · 홈페이지 연동형 웹페이지 (아이콘/링크 접속)
  • FastAPI 기반 웹 서비스로 구축, 메인 상담 페이지(/)와 홈페이지 삽입용 위젯 페이지(/widget)를 별도 제공
  • 위젯은 전체화면(100vh)·테두리 없는 형태로, 기존 사하구청 홈페이지에 iframe / 링크 / 아이콘 버튼으로 손쉽게 연동 가능
  • 위젯 임베딩 출처 제한(CORS)으로 공식 도메인에서만 안전하게 호출

반응형 UI/UX — PC·모바일 최적화 충족

요구 · PC 및 모바일 환경 최적화
  • 반응형 레이아웃: 데스크톱(≥768px)은 카드형 중앙 정렬, 모바일(≤767px)은 전체화면 전환
  • 사용자 편의 기능: 다크 모드, 글자 크기 조절(접근성), 음성 읽기(TTS), 마스코트 캐릭터 상태 표시
  • 설정값은 브라우저에 저장되어 재방문 시 유지

2. 주요 기능

① 데이터 수집 및 구조화 (Data Pipeline)

요구 · 증분 크롤링 — 최초 1회 전수 수집 후, 변경된 페이지만 주기적 감지·업데이트하여 서버 부하 최소화 고도화
  • 각 페이지의 ETag / Last-Modified를 저장하고, 다음 회차에 If-None-Match·If-Modified-Since 조건부 GET으로 요청
  • 응답을 4가지로 분기 처리 — 304(변경 없음, 본문 다운로드 생략) / 200(변경분만 재처리) / 404(삭제 페이지 DB 정리) / 일시 오류(보존·재시도)
  • 변경 감지된 페이지만 청크 재생성·재임베딩 → 네트워크·연산 부하 최소화
  • 매일 새벽 3시 자동 실행(스케줄러)으로 "주기적 감지·업데이트" 충족
요구 · AI 자동 태깅 — LLM으로 메타데이터(카테고리·부서·키워드 등) 자동 추출·태깅 충족
  • LLM이 각 문서에서 서비스 유형(민원·복지·세금·교통 등), 담당 부서명, 대상(노인·아동·장애인 등), 핵심 키워드, 한 줄 요약을 자동 추출
  • 5건씩 묶어 1회 호출하는 배치 태깅으로 처리 효율화, 실패 시 단건 폴백으로 안정성 확보

② 하이브리드 검색 (Hybrid Retrieval) 고도화

요구 · 2단계 검색 — ①태깅된 메타데이터로 탐색 범위 축소 → ②축소 범위 내 의미(Vector) 검색으로 정확도 향상
  • 1차 필터링: 질문에서 카테고리·서비스유형을 감지해 해당 범위로 탐색 대상 축소
  • 2차 정밀 검색: 축소 범위 내 벡터 의미 검색(pgvector) 수행
  • (추가) 3차 키워드 보강: BM25 키워드 점수를 가중 합산해 재랭킹 → 고유명사·숫자 질의 정확도 향상
  • 필터 결과가 부족하면 자동으로 전체 범위로 폴백하여 누락 방지

③ 멀티턴(Multi-turn) 대화 처리 충족

요구 · 심층 의도 파악(모호한 질문 시 역질문) + 문맥 유지(이전 대화 기억)
  • 역질문: 질문이 모호하면 바로 답을 끝내지 않고 선택지를 제시하며 되묻도록 설계(예: "1)노인복지 2)아동복지 3)장애인복지 …"). 전용 신호 토큰 + 키워드 폴백으로 이중 감지
  • 문맥 유지: 직전 대화 이력을 LLM에 함께 전달하고, 검색어에도 합성하여 "그건/거기" 같은 지시 표현이 포함된 후속 질문도 자연스럽게 처리

④ 답변 생성 및 출처 명시 충족

요구 · 자연어 요약(행정용어 → 쉬운 언어) + 출처 검증(원문 URL을 카드/버튼 형태로 반드시 제시)
  • 자연어 요약: 행정 용어를 일반인이 이해하기 쉬운 말로 풀고, 핵심을 먼저 간결히 답하도록 프롬프트 규칙화
  • 출처 카드: 답변 근거가 된 홈페이지 원문을 클릭 가능한 카드(새 창 링크)로 제시. 카드에 제목·분야 배지 표시
  • 관련 출처가 마땅치 않을 때도 상위 근거 문서를 폴백 제시하여 "출처 반드시 제시"를 보장

3. 유의사항 준수

Fact-based — 사실 기반 (환각 방지) 충족

  • "제공된 참고자료에 있는 정보만 사용, 없는 내용은 추측·생성 금지"를 시스템 규칙으로 강제
  • 자료에서 답을 못 찾으면 솔직히 "찾지 못했습니다" 안내 + 대표전화(051-220-4000)·홈페이지 안내

Privacy — 개인정보 보호 충족

  • 이중 차단: 정규식(주민번호·전화·카드·이메일) + NER 모델(이름·주소 등 비정형) 으로 입력·출력 양단에서 개인정보 마스킹
  • 개인정보 입력 감지 시 저장하지 않으며 입력 자제를 안내. 대화 이력은 보관 기간 경과 후 자동 정리

Performance — 응답 속도 충족

  • 임베딩·NER·검색 인덱스를 서버 기동 시 사전 로딩하여 첫 응답 지연 제거
  • 증분 크롤링으로 갱신 부하를 최소화, 검색 인덱스는 메모리 캐싱
