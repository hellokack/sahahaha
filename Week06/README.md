## 2. (선택과제) 라이선스 비교 분석 및 선택 기준
* **MIT License:** 가장 관대한 라이선스 중 하나로, 상업적 이용과 수정이 자유롭고 저작권 표시만 하면 됩니다. 초기 오픈소스 생태계 확장에 매우 유리합니다.
* **Apache License 2.0:** MIT와 비슷하지만 특허권 관련 조항이 명시되어 있어, 기업 간 협업이나 특허 이슈가 얽힐 수 있는 대규모 프로젝트에 더 안전합니다.
* **선택 기준:** 본 AI 민원 챗봇 프로젝트는 공공의 목적을 띠며 최대한 많은 개발자의 기여와 확장을 유도하기 위해, 가장 제약이 적고 널리 쓰이는 **MIT License**를 채택하는 것이 적합하다고 판단했습니다.

## 3. (선택과제) Inner Source 도입 로드맵 (조직 내부 적용)
프로젝트 팀 내부의 개발 문화를 오픈소스처럼 개방적이고 투명하게 만들기 위한 로드맵입니다.
* **1단계 (기반 구축):** 팀 내 모든 코드를 하나의 중앙 저장소로 통합하고, 누구나 열람할 수 있도록 접근 권한을 투명하게 개방합니다.
* **2단계 (표준화):** 사전에 작성한 `CONTRIBUTING.md`와 PR 템플릿을 도입하여, 다른 파트(예: 백엔드 팀원이 프론트엔드 코드에 기여)에서도 쉽게 코드를 기여할 수 있는 명확한 규칙을 확립합니다.
* **3단계 (문화 정착):** 코드 리뷰를 의무화하고 자유로운 의견 제안(RFC)을 장려하여, 부서 간 크로스 기여(Cross-contribution)가 자연스럽게 일어나는 환경을 조성합니다.
