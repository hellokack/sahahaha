# 📌 PRD: 사하구청 AI 상담사 프로토타입 개발

## 1. 프로젝트 개요 및 배경 (Overview & Background)
* [cite_start]**문제 정의 (Pain Points):** 최근 공공기관 홈페이지는 다양한 행정 정보와 민원 안내 서비스를 제공하고 있으나, 메뉴 구조가 매우 복잡하여 사용자가 필요한 정보를 직관적으로 찾기 어려운 한계가 존재합니다[cite: 22]. [cite_start]특히 주민들은 민원 신청 절차나 복지 정보 등을 빠르게 확인하고자 하지만, 기존의 키워드 매칭 중심 검색 방식은 접근성이 현저히 떨어집니다[cite: 23].
* [cite_start]**해결 방안 및 목표:** 사하구청 공식 홈페이지의 실제 행정 데이터를 기반으로, 사용자의 자연어 질의를 이해하고 필요한 정보를 즉시 제공하는 'AI 상담사 프로토타입'을 개발합니다[cite: 24]. [cite_start]비정형 행정 데이터를 구조화하고 하이브리드 검색 엔진을 도입하여, 주민들이 보다 편리하게 행정 서비스를 이용할 수 있는 지능형 환경을 구축하는 것을 목표로 합니다[cite: 25].

## 2. 타겟 사용자 (Target Audience)
* [cite_start]**주요 사용자:** 사하구청 홈페이지를 방문하여 행정, 민원, 복지 등의 정책 정보를 빠르고 정확하게 확인하고자 하는 **사하구 주민 및 일반 민원인**[cite: 23].

## 3. 핵심 요구사항 및 기능 명세 (Core Features)

### 3.1. 공공데이터 수집 및 구조화 (Data Pipeline)
* [cite_start]사하구청 홈페이지의 최신 행정 정보를 크롤링하여 데이터베이스화합니다[cite: 62].
* [cite_start]최초 전체 데이터 수집 이후, 증분 크롤링 기법을 도입하여 변경되거나 업데이트된 정보만 주기적으로 동기화합니다[cite: 63].
* [cite_start]LLM을 활용한 자동 태깅(Auto-tagging) 시스템을 구축하여 수집된 데이터에 '부서, 주제, 키워드' 등의 메타데이터를 자동으로 부여하고 비정형 데이터를 구조화합니다[cite: 64, 65, 66, 67].

### 3.2. 하이브리드 검색 시스템 (Hybrid Retrieval)
[cite_start]검색의 정확도와 속도를 극대화하기 위해 2단계 검색 아키텍처를 설계합니다[cite: 70].
* [cite_start]**1단계 (필터링):** 메타데이터(부서/주제 등)를 기반으로 탐색 범위를 일차적으로 축소합니다[cite: 71, 72, 73].
* [cite_start]**2단계 (의미 검색):** Vector DB를 활용하여 사용자의 질문과 가장 의미적으로 유사한 문서를 탐색(Vector Search)하여 할루시네이션(거짓 정보)을 최소화합니다[cite: 74, 75, 76].

### 3.3. 멀티턴 대화 처리 (Multi-turn Dialogue)
[cite_start]단발성 질의응답을 넘어, 실제 상담사와 대화하는 듯한 연속적인 사용자 경험을 제공합니다[cite: 78].
* [cite_start]이전 대화의 문맥(Context)을 기억하고 유지하여 자연스러운 후속 질문을 처리합니다[cite: 79].
* [cite_start]사용자의 질문이 모호하거나 정보가 부족할 경우, AI가 역질문을 제공하여 의도를 명확히 파악합니다[cite: 80, 81].

### 3.4. 사용자 친화적 답변 생성 및 출처 제공
* [cite_start]검색된 전문적인 행정 용어를 LLM이 일반인의 눈높이에 맞추어 이해하기 쉬운 자연어로 요약 및 가공하여 제공합니다[cite: 83, 84].
* [cite_start]신뢰성 확보를 위해 모든 답변의 근거가 되는 사하구청 원본 홈페이지 URL을 카드형 UI 또는 버튼 형태로 반드시 함께 표기합니다[cite: 85, 86].

## 4. 비기능 요구사항 및 검증 계획 (Non-functional & Testing)
* [cite_start]**보안성 (Security):** 사용자가 대화창에 민감한 개인정보를 입력할 경우, 이를 식별하여 데이터베이스에 저장되거나 AI 모델 학습에 사용되지 않도록 즉시 마스킹 및 차단 조치합니다[cite: 58, 95].
* [cite_start]**정확도 검증:** 실제 사하구청에 자주 인입되는 민원 질문 사례(FAQ)를 데이터셋으로 활용하여 AI 상담사의 응답 정확도를 평가합니다[cite: 91].
* [cite_start]**사용성 테스트:** 반응형 웹 UI/UX를 구현하고 응답 속도 지연을 최소화하여 사용자 편의성을 검증합니다[cite: 93].

## 5. 기술 스택 (Tech Stack)
* [cite_start]**Language:** Python [cite: 31]

* [cite_start]**Data & DB:** Pandas, Vector DB (FAISS 또는 Chroma) [cite: 35, 37]

## 6. 기대 효과 (Expected Impact)
* [cite_start]주민들이 복잡한 탐색 과정 없이 자연어 질의만으로 24시간 즉시 필요한 행정 정보를 얻을 수 있어 **대민 서비스 질이 획기적으로 향상**됩니다[cite: 99].
* [cite_start]단순/반복적인 민원 문의를 AI가 1차적으로 처리함에 따라, 구청 공무원들의 **행정 업무 피로도 감소 및 효율성 증대**를 기대할 수 있습니다[cite: 100].
* [cite_start]성공적인 프로토타입 구현 시, 타 지자체 및 공공기관 홈페이지에도 즉시 확장 적용할 수 있는 **범용적인 공공 AI 서비스 모델**로 발전할 수 있습니다[cite: 101].
