# Sahaha 프로젝트 기여 가이드라인 (Contributing Guidelines)

Sahaha 프로젝트에 관심을 가져주셔서 감사합니다! 본 프로젝트는 누구나 참여할 수 있는 오픈소스 프로젝트입니다. 기여를 원하시는 분들은 아래 가이드라인을 확인해 주시기 바랍니다.

## 🚀 기여 방법

1. **이슈(Issue) 등록:** 버그를 발견했거나 새로운 기능을 제안하고 싶다면, 먼저 Issue를 생성하여 논의해 주세요.
2. **포크(Fork) 및 브랜치 생성:** 프로젝트를 Fork한 후, 새로운 브랜치를 생성하여 작업을 진행합니다. (`feat/이슈번호-기능명` 또는 `bugfix/이슈번호-버그명`)
3. **코드 작성 및 테스트:** 코드를 작성하고, 기존 기능에 영향을 주지 않는지 반드시 로컬 테스트를 진행해 주세요. (테스트 커버리지 80% 이상 권장)
4. **Pull Request (PR) 생성:** 작업이 완료되면 `main` 브랜치를 향해 PR을 생성합니다. PR 본문에 어떤 Issue를 해결했는지 상세히 적어주세요.

## 💻 코드 컨벤션 (Code Convention)
* 커밋 메시지는 `[feat]`, `[fix]`, `[docs]`, `[chore]` 등의 머리말을 사용합니다.
* 모든 변경 사항은 CI 파이프라인(GitHub Actions)의 테스트를 통과해야 병합될 수 있습니다.
